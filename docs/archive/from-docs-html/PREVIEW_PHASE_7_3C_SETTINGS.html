JSDW AI Chat Settings

Core System

These options keep the assistant’s engine running. Turning them off can break chat, updates, or saving until you turn them back on.

Plugin enabled

Turns the whole assistant off or on. When off, nothing from this plugin runs.

REST API

Lets the chat talk to your site in the background so questions and answers can work.

Recommended: Enabled

WP-Cron

Lets WordPress run tasks on a schedule; the assistant may use this for upkeep (for example refreshing content).

Recommended: Enabled

Background processing queue

Runs heavy work in the background so your pages stay fast while content is prepared for answers.

Recommended: Enabled

Store conversations

Allows chats to be saved on your server when your other choices allow it (see “Save conversation transcripts” in Chat Behavior).

Recommended: Enabled

Data & Processing

How your site’s content is scanned and prepared so the assistant can answer from it.

Indexing pipeline

Finds and prepares your site’s pages and posts so the assistant can answer from them.

Recommended: Enabled

AI features (when available)

Turns on extra AI-powered options when you have them set up elsewhere.

Auto reindex

When you change content, queues a refresh so answers can use the latest information.

A short name for which AI service or setup you are using (your host or docs may tell you what to enter).

Chat Behavior

Controls how the assistant responds to users.

Chooses how strictly answers stick to your site’s content and whether optional wording help is used.

Adjusts how formal, friendly, or short replies tend to be.

Clarification

If a question is unclear, the assistant may ask a quick clarifying question before answering.

Enable AI-assisted phrasing (when available)

May smooth or tighten how an answer is worded when your answer mode allows it.

Debug trace (chat)

Adds technical detail for site administrators who need to troubleshoot replies.

Store trace snapshots

Saves technical snapshots on the server to help support or debugging (uses storage).

Ignores very short messages until the visitor types at least this many characters.

Caps how long a single message can be to keep the site responsive and safe.

Save conversation transcripts

If enabled, full chats can be stored in your site’s database; turn off if you do not want that.

Access & Security

Controls who can access your assistant.

Allow public query endpoint

Anyone visiting the site could use the assistant from the front end when this is on; make sure that matches your privacy and moderation plans.

Frontend Widget

Controls whether the chat widget appears on your site.

Front-end widget

Enables chat bubble features; you still choose below whether visitors actually see the bubble.

Recommended: Enabled

Widget enabled

When all requirements are met, visitors see the chat bubble on your site.

Picks which corner the bubble uses instead of the default from Design Studio.

Words shown on the closed chat button (for example “Chat” or “Help”).

Greeting visitors read when they open the chat.

Hint text inside the box where visitors type (before they start typing).

Show sources

When an answer cites your content, visitors can see where it came from.

Allow reset conversation

Lets visitors clear the chat and start over.

Admin debug UI

Shows advanced options in the bubble for administrators who manage the site.

Auto footer injection

If you did not add the chat shortcode to a page, the plugin can still load the bubble from your site footer.

Logging

A technical diary the plugin can write to help diagnose issues.

Logging enabled

Records technical events in a log to help track down problems.

Mirror WP_DEBUG

When WordPress debug mode is on, matches how detailed the log is.

For example, “info” and above includes routine messages; “error” means only serious problems.

Data & Cleanup

This action is irreversible.

Remove data on uninstall

When you uninstall the plugin, this removes its stored data from the database; you cannot undo that later.

Sources (reference)

Fine-grained rules for which pages and URLs feed the assistant live elsewhere. Use Source Registry and indexing tools to change them—not this screen.
