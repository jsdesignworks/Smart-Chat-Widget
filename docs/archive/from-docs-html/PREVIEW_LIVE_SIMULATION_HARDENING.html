Design Studio preview — live-simulation hardening

This note documents alignment between the admin Design Studio preview and the public widget’s open/close model, animation tokens, and typography feedback. No backend or widget intelligence behavior was changed; only preview shell JS/CSS and mirrored admin copies.

Root causes (prior behavior)

  • wrap.className = … reset the preview widget node each sync, stripping simulation classes (e.g. viewport/hidden overlays) until a later pass reapplied some of them — fragile and unlike the live DOM.
  • The live widget hides the chat window with display: none when the root lacks is-open; the preview used opacity: 0 and pointer-events: none, so layout and stacking differed from production.
  • The launcher only forced open; production toggles open/closed on repeated clicks.
  • Animation speed used --chat-anim-dur in preview while the widget sets --jsdw-anim-speed for keyframed opens.
  • Several preview elements used fixed px sizes, so font-size slider changes felt subtle on title, chips, and input.

What improved

  • is-open on #jsdw-pr-widget mirrors the live widget; closed panel uses display: none.
  • Position and animation classes applied with classList (no full className wipe).
  • jsdw-anim-slide|fade|pop + @keyframes in preview CSS match the public widget’s open motion.
  • --jsdw-anim-speed and derived --chat-font-size-lg / --chat-font-size-sm improve visible typography feedback.
  • Launcher click toggles the panel; aria-expanded, aria-controls, and aria-hidden on the panel.
  • Simulation bar (#jsdw-pr-sim-state) shows on/off chips for Panel, Badge, Quick replies, Typing, Timestamps, Branding, Sound.
  • Richer default welcome and user sample copy; user bubble can show timestamps when enabled.
  • Badge uses a light pulse when enabled so the toggle is obvious in preview.
  • Midnight theme panel shadow selector tied to .is-open instead of removed is-closed classes.

Validation checklist (WordPress admin)

  • Launcher opens and closes the panel repeatedly; close button still closes.
  • Changing Open Animation (slide / fade / pop) and Animation Speed visibly affects the open transition.
  • Font size slider clearly moves title, messages, chips, input, and launcher label.
  • Toggles for badge, quick replies, typing, timestamps, branding, sound update preview and simulation chips.
  • Midnight (and other) themes still read correctly when the panel is open.
